## VapourSynth Script Generation

### Example Generated Script (Theatre Mode Enabled)

```python
import sys, site, os
sys.path.append(site.getusersitepackages())
plugin_path_win = os.path.join(os.getenv('APPDATA') or '', 'VapourSynth', 'plugins64')
sys.path.append(plugin_path_win)

import vapoursynth as vs
core = vs.core
core.num_threads = 8
core.max_cache_size = 4096  # MB, reserve 20% VRAM for FFmpeg
import havsfunc as haf

# Source
video = core.ffms2.Source(source='C:\\Video\\tape001.avi')

# ===== THEATRE MODE: Chroma Phase Correction (Hardware-Accurate) =====
print('[Theatre Mode] Applying chroma correction: vhs_composite preset (X=0.5px, Y=0.0px)')

# Hardware-accurate chroma alignment (replicates analog chipset processing)
def chroma_phase_correct(clip, shift_x_px=0.0, shift_y_px=0.0):
    fmt = clip.format
    if fmt is None or fmt.color_family != vs.YUV:
        return clip  # Skip if not YUV
    
    # Split Y, U, V planes
    y = core.std.ShufflePlanes(clip, planes=0, colorfamily=vs.GRAY)
    u = core.std.ShufflePlanes(clip, planes=1, colorfamily=vs.GRAY)
    v = core.std.ShufflePlanes(clip, planes=2, colorfamily=vs.GRAY)
    
    # Subpixel shift chroma planes (zimg bicubic resampling)
    u_shifted = core.resize.Bicubic(u, u.width, u.height, src_left=-shift_x_px, src_top=-shift_y_px)
    v_shifted = core.resize.Bicubic(v, v.width, v.height, src_left=-shift_x_px, src_top=-shift_y_px)
    
    # Recombine planes
    out = core.std.ShufflePlanes([y, u_shifted, v_shifted], planes=[0, 0, 0], colorfamily=vs.YUV)
    
    # Preserve original format
    if out.format.id != clip.format.id:
        out = core.resize.Bicubic(out, format=clip.format.id)
    return out

video = chroma_phase_correct(video, shift_x_px=0.5, shift_y_px=0.0)
print('   [OK] Chroma phase correction applied')

# Theatre Mode: Standard Progressive Deinterlacing
video = haf.QTGMC(video, Preset='Slow', TFF=True, FPSDivisor=2)  # 60i → 30p
print('[Theatre Mode] Standard deinterlacing: Progressive output')

# [... other filters ...]

# ===== THEATRE MODE: Level Adjustment =====
print('[Theatre Mode] Adjusting levels: Black=0.100, White=0.850, Sat=1.20x')

# Adjust black/white points (expand dynamic range)
video = core.std.Levels(video, min_in=25, max_in=217, min_out=0, max_out=255, planes=0)
# Boost saturation for faded tapes
video = core.std.Expr(video, ['', f'x 128 - 1.2 * 128 +', f'x 128 - 1.2 * 128 +'])
print('   [OK] Level adjustment applied')

# Ensure final output is YUV420P8 (required for Y4M pipe)
if video.format.id != vs.YUV420P8:
    video = core.resize.Bicubic(video, format=vs.YUV420P8, matrix_s='709')

video.set_output()
```

## Settings Integration

Theatre Mode expects these keys in the options dict:

### Required Settings
```python
{
    "theatre_mode_enabled": False,          # Master enable switch
    "chroma_correction_enabled": False,     # Enable chroma phase correction
    "chroma_preset": "laserdisc",           # Preset: laserdisc/vhs_composite/svhs/hi8/betamax/custom
    "chroma_shift_x_px": 0.25,             # Horizontal chroma shift (pixels)
    "chroma_shift_y_px": 0.0,               # Vertical chroma shift (pixels)
    "deinterlace_variant": "standard",      # Variant: standard/bob/keep_interlaced
    "apply_level_adjustment": False,        # Enable level adjustment
    "black_point": 0.0,                     # Black level (0.0-1.0)
    "white_point": 1.0,                     # White level (0.0-1.0)
    "saturation_boost": 1.0,                # Saturation multiplier (1.0-2.0)
}
```

These settings are already added to restoration_settings.json in v4.0.

## Known Limitations

1. **VapourSynth Required:** Theatre Mode auto-profiling requires VapourSynth Python module (not just vspipe). May need to handle ImportError gracefully.

2. **Preset Validation:** If chroma_preset is invalid or misspelled, get_preset() returns "custom" with 0.0/0.0 values. GUI should validate preset names.

3. **Level Adjustment Precision:** Black/white point uses std.Levels which operates on 8-bit scale internally. For 10-bit+ sources, may have slight rounding.

4. **Saturation Boost Expression:** Uses std.Expr with simple linear scaling. More sophisticated algorithms (HSV conversion) could be added later.

## Performance Impact

Based on testing expectations:

- **Chroma Correction:** ~2-5% slower (bicubic resize on 2 planes)
- **Bob Deinterlacing:** 2x processing time (doubles frames)
- **Level Adjustment:** ~1-2% slower (per-pixel expressions)
- **Overall Theatre Mode:** ~5-10% slower than Normal Mode

VRAM usage unchanged (no additional buffers needed).
